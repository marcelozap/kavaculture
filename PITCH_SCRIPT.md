# Kava Culture Pitch Script

## 15-Second Version

You already have a strong brand website. This is the next layer that helps guests actually order. It gives each Kava Culture location a cleaner mobile experience for featured drinks, tipping, and phone checkout without replacing the current site.

## 30-Second Version

What I built is not meant to replace KavaCulture.com. Your current site already does a great job with brand, locations, and story. This is more of an upgrade layer for action. Once someone already wants to order, this gives them a smoother phone-first experience where they can choose their location, browse featured drinks, tip on their phone, and move through a cleaner ordering flow that still feels like Kava Culture.

## 60-Second Version

The idea here is not to redesign your whole website. You already have a strong brand presence online. What I wanted to explore was what happens after a guest is interested and ready to do something. This concept is a mobile ordering and guest-experience layer for each Kava Culture location. It keeps the visual tone and menu language, but makes the experience more actionable. A guest can choose a location, see featured drinks for that store, move through a simplified menu, add gratuity, and check out on their phone. It also creates room for each location to highlight its own drinks, promotions, and events while still keeping a consistent Kava Culture feel across stores.

## If They Ask "Why Do We Need This If We Already Have A Website?"

The current site is strong for discovery and brand. This is meant for the next step, when a guest is actually ready to order, tip, or engage with a specific location. It is more of an operational and guest-experience upgrade than a new marketing site.

## If They Ask "What Is The Main Benefit?"

The main benefit is a smoother guest experience on mobile. It makes ordering clearer, helps each location feature the right drinks, and gives Kava Culture a more modern action-oriented flow without replacing what already works.

## If They Ask "Is This A Replacement?"

No. The best version of this would work alongside the current site as an ordering layer, subdomain, or location-based mobile experience.

## If They Ask "How Would It Go Live?"

It could live as:

- an order subdomain
- a location ordering page
- an add-on flow connected to the current website

The production version would use real menu data, real store logic, and a real hosted checkout.
